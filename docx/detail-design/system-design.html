2  データフロー
方向 種別 名称 ペイロード 説明
FE → Rust invoke start_timer { minutes: u32 } タイマー開始。設定分数を渡す
FE → Rust invoke pause_timer なし タイマーを一時停止する
FE → Rust invoke resume_timer なし 一時停止中のタイマーを再開する
FE → Rust invoke reset_timer なし タイマーを初期状態に戻す
FE → Rust invoke play_alarm なし アラーム音の再生を開始する
FE → Rust invoke stop_alarm なし アラーム音を停止する
Rust → FE emit timer_tick { remaining_sec: u32 } 1秒ごとに残り秒数を送信する
Rust → FE emit timer_finished なし 残り時間が0になったことを通知する

lit = Math.ceil( remainingSec / totalSec × 30 )

点灯 19 消灯 11

6  非機能要件への対応
要件 設計上の対応
タイマー誤差 ±1秒以内 Rustの tokio::time::interval を使用。JavaScriptの setInterval は使わない。Rustからイベントで通知し、フロントは表示のみを担当する
低CPU使用率 Tauriはシステム搭載のWebViewを利用するためChromiumを内包しない。Rustの非同期タイマーはCPUをブロックしない
長時間動作の安定性 Rustのメモリ安全性によりメモリリークを防止。フロントエンドのイベントリスナーはアンマウント時に unlisten() で解除する
クロスプラットフォーム Tauriが各OSのWebViewとネイティブAPIを抽象化。音声は rodio、通知は tauri-plugin-notification が各OS向けに対応
オフライン動作 外部APIへの通信なし。すべての処理はローカルで完結する
