# 技術選定

## 1. 選定方針

- クロスプラットフォーム（Windows / macOS / Linux）で動作すること
- オフライン環境で完結すること
- 軽量かつ低CPU使用率で動作すること
- 開発・保守コストを最小化すること

---

## 2. アプリケーションフレームワーク

| 項目 | 選定 | 理由 |
|---|---|---|
| フレームワーク | **Tauri v2** | Rust製のデスクトップアプリフレームワーク。Electronと比較してバンドルサイズが小さく（~10MB以下）、メモリ使用量が少ない。OSネイティブのWebViewを使用するためインストール不要 |
| フロントエンド | **React 18 + TypeScript** | コンポーネント単位での状態管理がタイマーUIに適合。型安全性により実装ミスを防止 |
| バックエンド | **Rust（Tauri コア）** | タイマー精度・通知・アラーム音はRustで実装し、±1秒以内の精度を担保 |

### Tauri vs Electron 比較

| 比較項目 | Tauri v2 | Electron |
|---|---|---|
| バンドルサイズ | ~8MB | ~100MB+ |
| メモリ使用量 | 低（OSのWebView利用） | 高（Chromium内包） |
| CPU使用率 | 低 | 中〜高 |
| オフライン動作 | ○ | ○ |
| クロスプラットフォーム | ○ | ○ |

---

## 3. フロントエンド技術スタック

| 用途 | ライブラリ / ツール | バージョン | 理由 |
|---|---|---|---|
| UIフレームワーク | React | 18.x | コンポーネント設計・状態管理に適合 |
| 言語 | TypeScript | 5.x | 型安全。タイマー状態の型定義に有効 |
| スタイリング | CSS Modules | — | スコープ付きCSS。外部依存なし |
| ビルドツール | Vite | 5.x | 高速HMR。Tauriと公式統合済み |
| 状態管理 | React `useState` / `useReducer` | — | 規模に対して外部ライブラリ不要 |
| タイムゾーン | `Intl.DateTimeFormat` | Web標準 | ライブラリ不要。ブラウザ標準APIで完結 |

---

## 4. バックエンド（Rust）技術スタック

| 用途 | クレート | 理由 |
|---|---|---|
| タイマー精度 | `tokio` | 非同期ランタイム。精度の高いインターバル処理 |
| アラーム音再生 | `rodio` | クロスプラットフォーム対応の音声再生 |
| デスクトップ通知 | `tauri-plugin-notification` | Tauri公式プラグイン。OS通知に対応 |
| フロントエンド連携 | `tauri::command` | Rust関数をTypeScriptから呼び出す仕組み |

## 6. 選定外の技術と理由

| 候補 | 不採用理由 |
|---|---|
| Electron | バンドルサイズ・メモリ消費が過大。本アプリの規模に不釣り合い |
| Flutter | Dartの学習コストが高い。Webアセット流用不可 |
| NW.js | メンテナンスが低調。Tauriと比較して優位性がない |
| Zustand / Redux | 状態量が少なく、`useState` で十分対応可能 |
| Day.js / date-fns | タイムゾーン表示のみが目的であり `Intl` APIで代替可能 |
